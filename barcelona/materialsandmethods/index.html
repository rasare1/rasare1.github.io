Materials and Methods

Environment Setup

To analyze air quality data effectively, we established a robust data science environment using the following tools:

  • R Language The core statistical programming language used for data processing.
  • RStudio Our primary Integrated Development Environment (IDE).
  • Tidyverse A collection of R packages (like dplyr and ggplot2) designed for data science.
  • Openair A specialized library for the analysis of air pollution data.

Methodology and Workflow

1. Data Cleaning and Filtering

We started by cleaning the raw data because the original files were messy and contained information we did not need. Using tools like Tidyverse, we filtered out the extra data and organized everything so the computer could perfectly understand the specific dates and hours of every record.

2. Table Formatting and Merging

Next, we focused on fixing the table formats. We used a special script to turn technical sensor codes into easy-to-read columns, such as wind speed or the levels of each pollutant gas. Once the data was ready, we merged two different databases—one with weather info and one with pollution levels—into a single master table. This was essential to analyze how weather directly influences the quality of the air we breathe.

3. Scientific Visualization

Finally, we used the openair library to transform all those numbers into the charts you see on this website. We created visualizations that show how pollution changes depending on the time of day or the month of the year, heatmaps to detect moments with the highest nitrogen dioxide, and wind analysis to discover which direction the most polluted air comes from. This whole process allowed us to turn a long list of numbers into visual conclusions that are very easy to understand.

Data Source

Respiratory infection data and air quality records were obtained from the Information System for the Surveillance of Infections in Catalonia (SIVIC), covering Barcelona from 2012-2025.
